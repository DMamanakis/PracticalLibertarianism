## Purpose

This repository is the canonical source for the Libertarian Framework project.

The goal of this project is not to advocate for a political party, ideology, or
personality.

The goal is to produce a comprehensive, intellectually honest framework that:

- Accurately describes reality as it exists today.
- Identifies where existing systems succeed.
- Identifies where existing systems fail.
- Explains why those failures occur.
- Presents practical, constitutional, and logically consistent proposals for
  improvement.

The project is intended to be useful to readers regardless of political
affiliation.

## Fundamental Principle

This project is built upon one overriding principle:

> **Reality takes precedence over ideology.**

No political philosophy, including Libertarianism, is permitted to override
observable facts.

- Evidence always supersedes opinion.
- Truth is preferred over persuasion.
- Accuracy is preferred over rhetoric.

## Project Philosophy

This project intentionally separates three independent concepts.

### Layer 1 — Reality

Describe the world exactly as it exists today.

Do not describe the world as it should be.

Document:

- Existing laws
- Existing institutions
- Existing constitutional doctrine
- Existing economic systems
- Historical facts
- Current data
- Judicial precedent
- Practical outcomes

This layer must remain descriptive.

### Layer 2 — Analysis

Analyze why systems succeed or fail.

Explain:

- Incentives
- Unintended consequences
- Constitutional conflicts
- Economic effects
- Historical causes
- Strengths
- Weaknesses

Analysis must distinguish clearly between:

- Facts
- Interpretation
- Opinion

### Layer 3 — Proposal

Only after Reality and Analysis are complete may reforms be proposed.

Every proposal must answer:

1. Why is change necessary?
2. What problem is being solved?
3. What evidence supports the change?
4. Is the proposal constitutional?
5. Could it realistically be implemented?
6. What unintended consequences might exist?

## Constitutional Foundation

This project treats the Constitution of the United States as the governing legal
framework of the Republic.

The Constitution shall never be dismissed merely because it is inconvenient.

When discussing constitutional questions:

- Distinguish constitutional text from statute.
- Distinguish statute from regulation.
- Distinguish regulation from agency policy.
- Distinguish court interpretation from constitutional language.

Never conflate these concepts.

Whenever possible, identify:

- Constitutional Article
- Section
- Clause
- Amendment
- Relevant Supreme Court decisions

## Intellectual Honesty

Contributors **MUST**:

- Follow evidence wherever it leads.
- Acknowledge uncertainty.
- Admit conflicting evidence.
- Present opposing viewpoints fairly.
- Distinguish facts from opinions.
- Distinguish verified information from speculation.

Never intentionally omit significant evidence because it is politically
inconvenient.

## Required Definitions

The following words have precise meanings within this repository.

### Fact

A statement supported by verifiable evidence.

### Opinion

A subjective conclusion.

Opinions are permitted.

Opinions must never be presented as facts.

### Proposal

A recommendation for future action.

Proposals are never descriptions of existing reality.

### Constitutional

Consistent with the Constitution as written and interpreted through legitimate
constitutional processes.

### Unconstitutional

In conflict with constitutional authority or constitutional limitations.

This determination must always explain the constitutional basis.

### Liberty

The maximum practical freedom consistent with protecting the equal rights of
others.

### Government

A necessary institution whose legitimate purpose is the protection of rights and
the preservation of constitutional order.

## Citation Requirements (Mandatory)

- Every factual claim **SHALL** be traceable.
- Every statistic **SHALL** identify its source.
- Every quotation **SHALL** identify its source.
- Historical claims **SHALL** include citations whenever practical.
- Legal claims **SHALL** identify:
  - Constitution
  - Statute
  - Court case
  - Regulation

Whenever possible, prefer primary sources over secondary sources.

Preferred order:

1. Constitutional text
2. Federal and State statutes
3. Court decisions
4. Government publications
5. Academic research
6. Original historical documents
7. Reputable journalism

Avoid anonymous sources whenever possible.

## Editing Rules (Mandatory)

No contributor may:

- Change meaning without explicit justification.
- Remove important context.
- Remove citations.
- Introduce unsupported claims.
- Rewrite arguments into stronger or weaker positions.
- Insert political advocacy disguised as factual description.
- Simplify complex issues into slogans.
- Change constitutional terminology.
- Alter definitions without approval.

Every edit should be as small as reasonably possible.

- Preserve history.
- Preserve traceability.

## AI Contributor Rules

AI contributors **SHALL**:

- Ask for clarification rather than invent missing information.
- State uncertainty when appropriate.
- Never fabricate citations.
- Never fabricate quotations.
- Never fabricate statistics.
- Never fabricate legal authorities.
- Never fabricate historical events.
- Never assume user intent when repository evidence exists.
- Search repository documentation before creating new files.
- Prefer modifying existing material over creating duplicates.
- Explain significant edits.
- Produce reviewable changes.

## Repository Organization

- The repository shall maintain a single source of truth.
- Duplicate information should be eliminated whenever practical.
- Research should be linked rather than copied.
- Every chapter shall have a stable identifier.
- Every requirement shall have a stable identifier.
- Cross references should use identifiers rather than filenames whenever
  practical.

## Style Guide

- Prefer clarity over elegance.
- Prefer precision over persuasion.
- Prefer complete explanations over slogans.
- Avoid emotionally loaded language unless directly quoting historical material.
- Write for readers who disagree.
- Avoid sarcasm.
- Avoid ridicule.
- Assume the reader is intelligent.
- Explain terminology before using specialized language.

## Quality Standard

Contributors should behave like:

- Historians
- Constitutional scholars
- Systems engineers
- Economists
- Investigators

—not political campaign staff.

- Every assertion should survive hostile review.
- Every proposal should survive constitutional review.
- Every argument should survive logical review.

## Acceptance Criteria ("Lint Rules")

A contribution **SHALL NOT** be accepted if any of the following are true:

- [ ] Facts are unsupported.
- [ ] Citations are missing.
- [ ] Constitutional claims lack constitutional analysis.
- [ ] Opinions are presented as facts.
- [ ] Counterarguments are ignored.
- [ ] Definitions are inconsistent.
- [ ] Duplicate material has been introduced.
- [ ] Existing repository guidance has been violated.
- [ ] Significant edits lack explanation.
- [ ] Reasoning contains identifiable logical fallacies.

## Final Principle

This project exists to discover truth first.

Only after truth has been established should solutions be proposed.

If evidence disproves a previously held belief, the evidence wins.

If reality conflicts with ideology, reality wins.

The objective is not to defend Libertarianism.

The objective is to build the strongest constitutional framework that reality
can support.
